# APA@POS — Design Document

## Overview

A desktop Point of Sale application built with Python, Tkinter, and SQLite. Supports stock management, auto-generated barcode printing, order creation, and order history. Targets Windows (primary) and macOS.

---

## Tech Stack

| Layer         | Technology                          |
|---------------|-------------------------------------|
| UI            | Python 3.x + Tkinter                |
| Database      | SQLite 3 (via `sqlite3`)            |
| Barcode Gen   | `python-barcode` + `Pillow`         |
| Barcode Print | ZPL over USB to Zebra GX420D        |
| Packaging     | `pyinstaller`                       |

## Configuration (`config.py`)

```python
DB_PATH             = "pos.db"

# Zebra GX420D — USB only (no network)
# Windows: "\\\\.\\" + port name, e.g. "\\\\.\\COM3" or "\\\\.\\USB001"
# macOS:   "/dev/usb/lp0"
ZEBRA_USB_PATH      = "\\\\.\\USB001"

TAX_RATE            = 0.0          # e.g. 0.10 for 10%
CURRENCY_SYMBOL     = "$"
BARCODE_OUTPUT_DIR  = "assets/barcodes"
PAGE_SIZE           = 15           # rows per page in tables

# ── Receipt on checkout ───────────────────────────────────────────────────────
# "zebra"  — print a receipt label to the Zebra GX420D via USB
# "pdf"    — save a PDF receipt to RECEIPT_OUTPUT_DIR and open it
# "none"   — no receipt (silent checkout)
RECEIPT_MODE        = "none"
RECEIPT_OUTPUT_DIR  = "assets/receipts"  # used only when RECEIPT_MODE = "pdf"
```

## Printer — Zebra GX420D

| Property       | Value                                      |
|----------------|--------------------------------------------|
| Model          | Zebra GX420D (Direct Thermal)              |
| Connection     | USB                                        |
| Resolution     | 203 DPI                                    |
| Paper          | Direct Thermal Receipt Paper 4" × 574 ft  |
| Label width    | 4 inches = 812 dots @ 203 DPI             |
| Label height   | Variable — set per ZPL job                 |

ZPL template for a product label (4" wide, ~1.5" tall):
```zpl
^XA
^PW812
^FO30,20^A0N,35,35^FD{name}^FS
^FO30,70^BCN,80,Y,N,N^FD{barcode}^FS
^XZ
```

Sending to USB on Windows:
```python
with open(ZEBRA_USB_PATH, "wb") as f:
    f.write(zpl.encode("utf-8"))
```

On macOS:
```python
with open(ZEBRA_USB_PATH, "wb") as f:
    f.write(zpl.encode("utf-8"))
```
(Same code; only the path in `config.py` differs.)

## Barcode Auto-Generation

Barcodes are generated by the app when a product is created — the user never types one manually.

**Format:** `APA` prefix + zero-padded product ID + Luhn check digit, e.g. `APA000001C`

```python
# barcode_service.py
def generate_barcode_number(product_id: int) -> str:
    """Return a unique Code128-safe barcode string for a given product ID."""
    return f"APA{product_id:06d}"
```

The barcode string is stored in `products.barcode` and printed as Code128.

### 2. Stock Management Screen (`stock_view.py`)

```
┌──────────────────────────────────────────────────────────────┐
│  [Search: ________________________________] [Search]          │
├──────┬──────────────────┬──────────┬────────┬────────────────┤
│  ID  │  Name            │ Barcode  │ Stock  │  Price         │
├──────┼──────────────────┼──────────┼────────┼────────────────┤
│  1   │  Product A       │ APA000001│  50    │  $9.99         │
│  2   │  Product B       │ APA000002│   0    │  $14.99  ⚠     │
└──────┴──────────────────┴──────────┴────────┴────────────────┘
  [← Prev]  Page 1 of 3  [Next →]
  [Add Product] [Edit Selected] [Delete Selected] [Print Barcode] [Print Stock Report]
```

**Behaviours:**
- Search matches on name **or** barcode simultaneously (universal search).
- Table paginates at 15 rows; page resets on search change.
- Double-click a row → opens Edit modal.
- **Add Product**: modal with fields Name, Stock, Price. Barcode is auto-generated from the new product ID — not shown as an input field.
- **Edit Selected**: same modal pre-filled; barcode shown read-only.
- **Delete Selected**: confirms then removes (blocked if product has order history).
- **Print Barcode**: generates Code128 PNG via `barcode_service`, sends one ZPL label to Zebra GX420D via USB. One label per click.
- **Print Stock Report**: opens a preview modal (all products, sorted by name, columns: ID / Name / Barcode / Stock / Price / Value) with totals footer and a Print button.
- Rows where `stock = 0` are highlighted in a warning colour and a ⚠ indicator is shown.

## Data Flow — Checkout

```
User scans / types barcode (or selects by name)
      │
      ▼
Lookup product in DB → check stock > 0
      │  (stock = 0 → show warning, block add)
      ▼
Add to in-memory order list
      │
User clicks Checkout
      │
      ▼
BEGIN TRANSACTION
  INSERT INTO orders (total, created_at)
  For each item:
    INSERT INTO order_items (order_id, product_id, quantity, unit_price)
    UPDATE products SET stock = stock - quantity WHERE id = product_id
COMMIT
      │
      ▼
receipt_service.print_receipt(order, items)
      │  RECEIPT_MODE = "zebra" → ZPL to USB printer
      │  RECEIPT_MODE = "pdf"   → PDF saved + opened
      │  RECEIPT_MODE = "none"  → skip
      │  (error → show non-blocking warning; checkout already succeeded)
      ▼
Clear order UI → show confirmation dialog (order ID + total)
```

---

## Data Flow — Add Product & Print Barcode

```
User clicks [Add Product] → fills Name, Stock, Price → Save
      │
      ▼
INSERT INTO products (name, stock, price, barcode=NULL)
      │
      ▼
barcode_service.generate_barcode_number(new_id) → barcode string
UPDATE products SET barcode = ? WHERE id = new_id
      │
      ▼
[Print Barcode] clicked on that row
      │
      ▼
barcode_service.generate_barcode_png(barcode)  → PNG
      │
      ▼
zebra_service.build_product_zpl(name, barcode) → ZPL string
      │
      ▼
zebra_service.print_label_usb(zpl)             → write to USB path
```
